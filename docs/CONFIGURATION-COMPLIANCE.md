# DNSForge Configuration Compliance Engine

DNSForge v14.7.0 introduces a compliance engine that compares the rendered DNSForge BIND configuration with the deployed native BIND configuration.

## Commands

```bash
dnsforge config fingerprint
dnsforge config verify
dnsforge config drift
dnsforge config baseline show
dnsforge config baseline rebuild
dnsforge config repair --preview
```

## Status values

- `COMPLIANT`: deployed files match the rendered baseline.
- `DRIFTED`: at least one deployed file differs from the rendered baseline.
- `FAILED`: a critical resource is missing or the baseline cannot be evaluated.
- `WARNING`: reserved for non-blocking advisory findings.

## Safety

`repair --preview` is non-destructive. It only reports resources that would be restored or regenerated by a future remediation workflow.
